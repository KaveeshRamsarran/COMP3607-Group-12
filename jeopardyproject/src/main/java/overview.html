Jeopardy Game - Trivia Application

Project Overview

A comprehensive Jeopardy-style trivia game application built with Java, demonstrating SOLID principles and multiple design patterns. Features both console and GUI interfaces, multiple file format support, and comprehensive game reporting.

Key Features

  • Dual Interface: Console (JeopardyApp) and GUI (JeopardyAppGUI) versions
  • Multi-Format Support: Load questions from CSV, JSON, or XML files
  • Process Mining: Automatic event logging for game analysis
  • Rich Reporting: Generate reports in TXT, PDF, or DOCX formats
  • Score Tracking: Complete turn-by-turn history for all players

Architecture

The application is organized into several packages:

  • com.comp3607 - Core game logic and models
  • com.comp3607.parsers - File parsing strategies
  • com.comp3607.strategies - Answer validation strategies
  • com.comp3607.patterns - Design pattern examples

Design Patterns

Singleton Pattern
ProcessLog ensures single instance for event logging across the application
Factory Pattern
FileParserFactory creates appropriate parsers based on file type
Strategy Pattern
CategoryStrategy allows flexible answer validation and scoring algorithms

SOLID Principles

  • Single Responsibility: Each class has one clear purpose
  • Open/Closed: Open for extension (new parsers, strategies) without modification
  • Liskov Substitution: Strategy implementations are interchangeable
  • Interface Segregation: Focused interfaces (FileParser, CategoryStrategy)
  • Dependency Inversion: Depends on abstractions, not concrete classes

Getting Started

To run the application:

# Console version
mvn exec:java -Dexec.mainClass="com.comp3607.JeopardyApp"

# GUI version
mvn javafx:run

# Or build and run JAR
mvn clean package
java -jar target/jeopardy-game-1.0-SNAPSHOT.jar
